## 概要

Zigで形態素解析を開発しています。(全然途中です...)

https://github.com/fjnkt98/zarame

[unidic-cwj-202512](https://clrd.ninjal.ac.jp/unidic/back_number.html#unidic_cwj)を元にビルドした辞書をバイナリに同梱していて、現時点では単語の分かち書きだけができるようになっています。

> ```sh
> $ zig build run -Doptimize=ReleaseSafe
> 今日はいい天気ですね。
> Loaded 1027505 morphs, matrix rows=24082, cols=21375. Double array size is 2688901.
> BOS
> 今日
> は
> いい
> 天気
> です
> ね
> 。
> EOS
> ```

業務で日本語検索をしており、要素技術の理解を深めるために形態素解析器の自作をすることにしました。あとZig言語で何か作りたかった。

## 何故Zigなのか

まだ誰も作ってなさそうだったから(多分)。

調べた限り、[jaime](https://github.com/egegungordu/jaime)という日本語入力ツールが形態素解析と同様の辞書引きアルゴリズムをZigで実装していましたが、形態素解析器そのもののZig実装は無さそうでした。
これは作るしかないでしょう！というわけで偉大な先行実装([MeCab](https://github.com/taku910/mecab), [Kagome](https://github.com/ikawaha/kagome), [Lindera](https://github.com/lindera/lindera))を元にZigで作り始めたのがきっかけです。

## 実装済み / 未実装機能

現時点で下記を実装済みです。

- ダブル配列
- 共通接頭辞検索インデックス
- ラティスとビタビアルゴリズム
- 辞書のビルド・保存・バイナリ埋め込みと実行時ロード

下記は未実装です。形態素解析器としての最低限の機能すらまだありません...

- 複数辞書のサポート(IPADicや古いUniDic)
- 品詞情報やその他の単語情報(読みなど)の保持と出力
- 未知語処理
- 分かち書きした単語の語形変化処理

また、辞書のサイズがやたら大きかったり、ビルド時や実行時にメモリをやたら使用するのを解決したいです。データ構造の見直しやデバッグが必要そうです。

将来的にはC言語からも利用できるようにしたり、WASMとしてビルドしてブラウザ上で実行できるようにしてみたいと思います。
