備忘録。

## 環境

- macOS Tahoe 26.3
- Apple M3
- Tart 2.31.0

以下、`%`始まりのコマンドは macOS 上で、`$`始まりのコマンドは仮想環境上で実行することを意味する。

## Tart のインストール

Homebrew でインストールする。Quick Start に記載されている手順と同じものを実行する。

https://tart.run/quick-start/

```sh
% brew install cirruslabs/cli/tart
```

インストールできたか確認する。

```sh
% tart --version
```

## Rocky Linux 9 VM の作成

公式で Rocky Linux のイメージが提供されているのでこれを使う。

https://github.com/cirruslabs/linux-image-templates/pkgs/container/rocky

下記の例では`rocky`という名前を付けてVMを作成する。

```sh
% tart clone ghcr.io/cirruslabs/rocky:9 rocky
```

VMが作成されたか確認する。

```sh
% tart list
```

## リソース割り当ての確認・変更

`tart get`コマンドでVMに割り当てられているリソース等の設定を確認できる。

```sh
% tart get rocky
OS    CPU Memory Disk DiskFormat Size  Display  State
linux 4   4096   20   raw        1.668 1024x768 running
```

`tart set`コマンドで設定を変更できる。ただし、ディスクサイズは増やすことしかできない。

```sh
% tart set rocky --cpu=2 --memory=2048 --disk-size=20
```

## VMの起動

`tart run`でVMを起動できる。コンソールが起動し、ログインを求められる。初期値ではユーザ名・パスワード共に`admin`という値なのでそれを入力する。

```sh
% tart run rocky
```

`tart run`コマンドはフォアグラウンドで起動され、Ctrl+C等で終了するとVMも終了する。
`--no-graphics`オプションを付けることでコンソールの起動を省略できる。

SSHでの接続もできる。`tart ip`コマンドでローカルIPアドレスを取得できるので、これで接続する。

```sh
% ssh admin@$(tart ip rocky)
```

VMは起動しっぱなしで運用することが多いと思うので、バックグラウンドで起動しておくとよい。

```sh
% nohup tart run rocky --no-graphics &
```

VM一覧の確認は`tart list`でできる。

```sh
% tart list --source=local
```

終了するときは`tart stop`を使って終了できる。

```sh
% tart stop rocky
```

## パッケージのアップデート

パッケージをアップデートしておく。

```sh
$ sudo dnf check-update
$ sudo dnf update
```

## 各ミドルウェアのインストール

### Grafana

https://grafana.com/docs/grafana/latest/setup-grafana/installation/redhat-rhel-fedora/

GPGキーのインポート。

```sh
$ sudo dnf install wget

$ wget -q -O gpg.key https://rpm.grafana.com/gpg.key
$ sudo rpm --import gpg.key
```

設定ファイル作成。

```sh
$ sudo vim /etc/yum.repos.d/grafana.repo
# 下記を追記
```

```conf
[grafana]
name=grafana
baseurl=https://rpm.grafana.com
repo_gpgcheck=1
enabled=1
gpgcheck=1
gpgkey=https://rpm.grafana.com/gpg.key
sslverify=1
sslcacert=/etc/pki/tls/certs/ca-bundle.crt
```

Grafana をインストール。

```sh
$ sudo dnf install grafana
```

Grafana の起動とサービス登録。

```sh
$ sudo systemctl daemon-reload
$ sudo systemctl start grafana-server
$ sudo systemctl enable grafana-server.service
```

ポート3000で Grafana が起動するので、ホストのブラウザからダッシュボードにアクセスできる。

```sh
% open "http://$(tart ip rocky):3000/"
```

後始末。

```sh
$ rm -i gpg.key
```

### Prometheus

Prometheus 実行用ユーザの作成。

```sh
$ sudo useradd -r -M -c 'prometheus user' -s /sbin/nologin prometheus
$ id prometheus
```

Prometheus のインストール。ここでは下記の通りの設定でインストールする。

- `prometheus`, `promtool`の実行バイナリは`/usr/local/bin`ディレクトリ配下に配置する。
- 設定ファイルやストレージは`/etc/prometheus`配下に配置する。

```sh
$ wget https://github.com/prometheus/prometheus/releases/download/v3.5.1/prometheus-3.5.1.linux-arm64.tar.gz
$ ls -l prometheus-3.5.1.linux-arm64.tar.gz
$ sha256sum prometheus-3.5.1.linux-arm64.tar.gz
# cdb1de33d6d3ed73b4633cebedb70258c29e41bd132d287d387f322a046143d2

$ tar -xzvf prometheus-3.5.1.linux-arm64.tar.gz

$ sudo cp -i prometheus-3.5.1.linux-arm64/prometheus /usr/local/bin/prometheus
$ sudo chown root:root /usr/local/bin/prometheus
$ sudo cp -i prometheus-3.5.1.linux-arm64/promtool /usr/local/bin/promtool
$ sudo chown root:root /usr/local/bin/promtool

$ sudo mkdir /etc/prometheus
$ sudo cp -i prometheus-3.5.1.linux-arm64/prometheus.yml /etc/prometheus/prometheus.yml
$ sudo chown -R prometheus:prometheus /etc/prometheus/
```

サービス設定ファイルの作成。

```sh
$ sudo vim /etc/systemd/system/prometheus.service
# 下記を追記
```

```conf
[Unit]
Description=Prometheus Server
Documentation=https://prometheus.io/docs/introduction/overview/
Wants=network-online.target
After=network-online.target

[Service]
User=prometheus
Group=prometheus
Restart=on-failure
RestartSec=1
WorkingDirectory=/etc/prometheus
ExecStart=/usr/local/bin/prometheus --config.file=/etc/prometheus/prometheus.yml --web.enable-remote-write-receiver

[Install]
WantedBy=multi-user.target
```

サービス起動。

```sh
$ sudo systemctl daemon-reload
$ sudo systemctl start prometheus
$ sudo systemctl enable prometheus
```

ポート9090で Prometheus が起動するので、ホストのブラウザからダッシュボードにアクセスできる。

```sh
% open "http://$(tart ip rocky):9090/"
```

後始末

```sh
$ rm -rI prometheus-3.5.1.linux-arm64
$ rm -i prometheus-3.5.1.linux-arm64.tar.gz
```

### Grafana Alloy

公式手順に従ってインストールする。ただし、GPGキーの設定はGrafanaのインストール時に既に行っているため不要。

https://grafana.com/docs/alloy/latest/set-up/install/linux/#install-grafana-alloy-on-linux

```sh
$ sudo dnf install alloy
```

設定ファイルを更新する

```sh
$ sudo vim /etc/alloy/config.alloy
```

```
logging {
  level = "warn"
}

// OpenTelemetry Collector 設定
otelcol.receiver.otlp "default" {
  grpc {
    endpoint = "0.0.0.0:4317"
  }

  http {
    endpoint = "0.0.0.0:4318"
  }

  output {
    traces = [otelcol.processor.batch.default.input]
  }
}

otelcol.processor.batch "default" {
  output {
    traces = [otelcol.exporter.otlp.default.input]
  }
}

otelcol.exporter.otlp "default" {
  client {
    endpoint = "127.0.0.1:4319"
    tls {
      insecure = true
    }
  }
}

// Node Exporter 設定
prometheus.exporter.unix "node" {
  include_exporter_metrics = true
  disable_collectors       = ["mdadm"]
}

prometheus.scrape "metrics" {
  targets         = prometheus.exporter.unix.node.targets
  forward_to      = [prometheus.remote_write.default.receiver]
  scrape_interval = "15s"
}

// PostgresSQL 設定(PostgreSQLの設定は後述)
prometheus.exporter.postgres "postgresql" {
  data_source_names = ["postgresql://prometheus@localhost:5432/postgres?sslmode=disable"]

  autodiscovery {
    enabled           = true
    database_denylist = []
  }
}

prometheus.scrape "postgresql" {
  targets    = prometheus.exporter.postgres.postgresql.targets
  forward_to = [prometheus.remote_write.default.receiver]
}

// Prometheus への書き出しの設定
prometheus.remote_write "default" {
  endpoint {
      url = "http://localhost:9090/api/v1/write"
  }
  external_labels = { "node" = "rocky" }
}
```

その後起動する。

```sh
$ sudo systemctl start alloy
$ sudo systemctl enable alloy.service
```

### Grafana Tempo

GitHub Releases から rpm を入手してインストールする。

https://grafana.com/docs/tempo/latest/set-up-for-tracing/setup-tempo/deploy/locally/linux/

```sh
wget https://github.com/grafana/tempo/releases/download/v2.10.2/tempo_2.10.2_linux_arm64.rpm
ls -l tempo_2.10.2_linux_arm64.rpm
sha256sum tempo_2.10.2_linux_arm64.rpm
# ac7cb2a8b51cffd534b2ce88daca23df41c1607f61c62d0cef81933e2a6b9fac

sudo rpm -ivh tempo_2.10.2_linux_arm64.rpm
```

コンフィグファイルを設定する。

```sh
$ sudo vim /etc/tempo/config.yml
```

```yaml
stream_over_http_enabled: true
server:
  http_listen_port: 3200
  log_level: info

query_frontend:
  search:
    duration_slo: 5s
    throughput_bytes_slo: 1.073741824e+09
    metadata_slo:
        duration_slo: 5s
        throughput_bytes_slo: 1.073741824e+09
  trace_by_id:
    duration_slo: 5s

distributor:
  receivers:
    otlp:
      protocols:
        grpc:
          endpoint: "0.0.0.0:4319"

metrics_generator:
  registry:
    external_labels:
      source: tempo
      cluster: rocky
  storage:
    path: /var/tempo/generator/wal
    remote_write:
      - url: http://localhost:9090/api/v1/write
        send_exemplars: true
  traces_storage:
    path: /var/tempo/generator/traces

storage:
  trace:
    backend: local                     # backend configuration to use
    wal:
      path: /var/tempo/wal             # where to store the wal locally
    local:
      path: /var/tempo/blocks

overrides:
  defaults:
    metrics_generator:
      processors: [service-graphs, span-metrics, local-blocks] # enables metrics generator
      generate_native_histograms: both
```

Tempo を再起動する。 (rpm でインストールしたのでサービスは既に登録されている)

```sh
$ sudo systemctl restart tempo
```

### PostgreSQL 18

公式ページの手順に従ってインストールする。ただし、この通りに実行するとロケールがサーバのものを引き継いでしまうので、環境変数を用いて明示的に`--no-locale`を指定する。

https://www.postgresql.org/download/linux/redhat/

```sh
$ sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-aarch64/pgdg-redhat-repo-latest.noarch.rpm
sudo dnf install -y postgresql18-server

export PGSETUP_INITDB_OPTIONS='--no-locale --encoding=UTF8'
sudo -E /usr/pgsql-18/bin/postgresql-18-setup initdb

sudo systemctl enable postgresql-18
sudo systemctl start postgresql-18
```

監視用のユーザを作成し、`pg_monitor`の付与と認証設定を行う。監視用なのでローカルからのみのパスワード無しで接続できる設定にする。

```sh
sudo -u postgres psql
```

```sql
CREATE USER prometheus;
GRANT pg_monitor TO prometheus;

\q
```

```sh
sudo vim /var/lib/pgsql/18/data/pg_hba.conf
# 下記を追記(既存のhost allの設定より上に記載する必要がある)
# host    all             prometheus             127.0.0.1/32            trust
# host    all             prometheus             ::1/128                 trust
```

PostgreSQLを再起動して設定を反映させる。

```sh
sudo systemctl restart postgresql-18
```

prometheus ユーザで稼働統計情報を取得できることを確認しておく。

```sh
psql -U prometheus -h localhost postgres -c 'select usename, backend_type, query from pg_stat_activity;'
```
